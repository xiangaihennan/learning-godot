# Godot 学习提示词手册（按阶段 + 按步骤完整可复制）

> 目标：你只需要复制对应提示词发给我，就能按 [godot_2d_learning_plan.md](/Users/liuzeqi/Desktop/learning-godot/godot_2d_learning_plan.md) 一步一步推进。

## 0. 使用方法（先看这一段）

每一步都建议你按这个流程操作：

1. 打开计划文档，确认当前要做的步骤编号（例如第 7 步）。
2. 复制本手册里对应步骤的提示词，直接发给我。
3. 我给你：节点结构、代码、详细注释、Inspector 参数、验收步骤、排错方案。
4. 你在 Godot 里完成搭建并运行。
5. 把报错或现象贴给我，我继续带你修到通过验收。
6. 验收通过后，再进入下一步。

## 2. 阶段使用说明

### 阶段 A（步骤 1-9）

目标：先把“能玩”跑通，不追求美术与高级系统。  
做法：每步优先最小可运行，不提前做优化。

### 阶段 B（步骤 10-19）

目标：把“能玩”变成“有成长系统的游戏”。  
做法：强调数据驱动，减少硬编码。

### 阶段 C（步骤 20-23）

目标：扩内容、做手感、做流程闭环。  
做法：每加内容都要跑回归，防止旧功能回退。

### 阶段 D（步骤 24-30）

目标：工程化、可维护、可导出。  
做法：先做规范，再做发布检查。

### 阶段 E/F/G（步骤 31-38）

目标：长期扩展能力 + 学习沉淀。  
做法：把“经验”变成文档和配置，而不是只堆代码。

```text
我在做第N步时遇到问题。

- （完整粘贴）

- （粘贴）

我当前脚本：

请你：
1) 先判断是节点引用问题、生命周期问题、输入配置问题还是逻辑问题
```

```text
- 给二次验收步骤
```
